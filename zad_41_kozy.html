Rodzaje pętli

1) Pętla for

Konstrukcja: for (zainicjowanie_zmiennej; warunek_kończący_wykonywanie_pętli; zmiana_zmiennej) { // kod, który zostanie wykonany pewną ilość razy } Opis:

Pętla for wykonuje blok kodu określoną liczbę razy. Najpierw ustalana jest zmienna początkowa, potem sprawdzany warunek, a na końcu zwiększana (lub zmniejszana) zmienna sterująca.

2) Pętla while – z kontrolowanym wejściem

Konstrukcja: while (warunek) { // kod wykonywany dopóki warunek jest spełniony } Opis:

Pętla while najpierw sprawdza warunek, a dopiero potem wykonuje kod. Może nie wykonać się ani razu, jeśli warunek od początku jest fałszywy.

3) Pętla do...while – z kontrolowanym wyjściem

Konstrukcja: do { // ...fragment kodu który będzie powtarzany... } while (false); Opis:

Pętla do...while najpierw wykona instrukcję, a dopiero potem sprawdza warunek. Dzięki temu zawsze wykona się przynajmniej jeden raz – nawet jeśli warunek jest fałszywy.
